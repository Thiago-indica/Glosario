Glosario de Base de Datos


A

Administrador de Bases de Datos (DBA - Database Administrator): Profesional encargado de la gestión, mantenimiento, rendimiento y seguridad de una base de datos. Sus tareas incluyen el diseño, implementación, monitoreo y respaldo de la base de datos.

Atributo: Característica o propiedad que describe una entidad en un modelo de base de datos. Por ejemplo, si "Estudiante" es una entidad, "Nombre", "Apellido" y "DNI" podrían ser sus atributos.

B

Base de Datos (DB): Colección organizada de datos estructurados, típicamente almacenados y accesibles electrónicamente desde un sistema informático. Las bases de datos están diseñadas para facilitar el almacenamiento, la gestión y la recuperación de grandes volúmenes de información.

Base de Datos Relacional (RDB): Tipo de base de datos que organiza los datos en tablas (relaciones) que se interconectan mediante claves. Es el modelo más común para las bases de datos de aplicaciones comerciales.

C

Campo: Sinónimo de atributo o columna en una tabla de base de datos, representa una categoría específica de información.

Clave Ajena (Foreign Key - FK): Columna o conjunto de columnas en una tabla que hace referencia a la clave primaria de otra tabla. Establece la relación entre dos tablas, asegurando la integridad referencial.

Clave Primaria (Primary Key - PK): Una o más columnas en una tabla cuyos valores identifican de forma única cada fila (registro) de esa tabla. No puede contener valores nulos y cada valor debe ser único.

Columna: Sinónimo de atributo o campo en una tabla de base de datos.

Consulta: Solicitud de datos o información de una base de datos. Se realiza típicamente utilizando un lenguaje de consulta como SQL.

CRUD (Create, Read, Update, Delete): Acrónimo que describe las cuatro operaciones básicas que se pueden realizar sobre los datos en una base de datos (crear, leer, actualizar y eliminar).

D

Dato: Unidad individual de información. En el contexto de bases de datos, son los valores específicos almacenados en los campos de los registros.

DBMS (Database Management System - Sistema de Gestión de Bases de Datos): Software que permite a los usuarios definir, crear, mantener y controlar el acceso a la base de datos. Ejemplos incluyen MySQL, PostgreSQL, Oracle, SQL Server.

Diagrama de Entidad-Relación (DER / ERD - Entity-Relationship Diagram): Herramienta gráfica utilizada para modelar la estructura de una base de datos, mostrando las entidades (cosas o conceptos importantes), sus atributos y las relaciones entre ellas.

Diseño de Bases de Datos: Proceso de creación de la estructura de una base de datos, que incluye la definición de tablas, campos, relaciones y restricciones para satisfacer los requisitos de almacenamiento de datos de una aplicación.

E

Entidad: Objeto o concepto del mundo real sobre el que se desea almacenar información en una base de datos. En un DER, se representa típicamente como un rectángulo. Ejemplos: "Cliente", "Producto", "Pedido".

Esquema: La estructura lógica de una base de datos, que describe cómo se organizan los datos y las relaciones entre ellos. Incluye las tablas, campos, tipos de datos, claves y restricciones.

I

Índice: Estructura de datos que mejora la velocidad de las operaciones de recuperación de datos en una tabla. Similar al índice de un libro, permite encontrar filas de datos de forma más rápida.

Integridad de Datos: La exactitud, consistencia y fiabilidad de los datos a lo largo de su ciclo de vida. Se asegura mediante restricciones (claves primarias, foráneas, valores únicos).

N

Normalización: Proceso de organizar las columnas y tablas de una base de datos relacional para minimizar la redundancia de datos y mejorar la integridad de los datos. Se divide en formas normales (1NF, 2NF, 3NF, etc.).

P

Programación en SQL: Se refiere a la escritura de sentencias SQL para interactuar con una base de datos, incluyendo la creación de estructuras (DDL) y la manipulación de datos (DML).

R

Registro (Tupla / Fila): Una fila individual en una tabla de base de datos que contiene un conjunto completo de datos para una única instancia de una entidad.

Relación (en DER): Asociación o vínculo entre dos o más entidades. En un DER, se representa típicamente como un rombo. Describe cómo las entidades interactúan entre sí (ej., un "Cliente" realiza un "Pedido").

Relación (en Base de Datos Relacional): Se refiere a una tabla en el contexto de las bases de datos relacionales.

SQL (Structured Query Language): Lenguaje estándar, declarativo y procedimental, utilizado para gestionar y manipular bases de datos relacionales. Permite definir la estructura de la base de datos (DDL) y manipular los datos (DML).

T

Tabla: Estructura fundamental en una base de datos relacional, compuesta por filas y columnas, que organiza los datos de una entidad específica.
