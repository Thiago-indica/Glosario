Glosario de Sistemas Operativos


A

API (Application Programming Interface - Interfaz de Programación de Aplicaciones): Un conjunto de definiciones y protocolos que permiten que diferentes aplicaciones de software se comuniquen entre sí. El sistema operativo expone sus funcionalidades a las aplicaciones a través de APIs, permitiéndoles interactuar con el hardware y los servicios del SO.

B

Booting (Arranque): El proceso de iniciar una computadora. Involucra la carga inicial del sistema operativo desde el disco duro a la memoria RAM para que la computadora pueda empezar a funcionar.

Buffer: Un área de memoria temporal utilizada para almacenar datos mientras se están moviendo de un lugar a otro, o para compensar diferencias en la velocidad de transferencia de datos entre dispositivos.

C

Cola de Procesos: Una estructura de datos que mantiene el orden de los procesos que están esperando para ser ejecutados por el CPU o para acceder a algún recurso del sistema.

Concurrencia: La capacidad de un sistema para manejar múltiples tareas o procesos "al mismo tiempo". Esto no siempre significa ejecución paralela real, sino que pueden estar progresando simultáneamente, intercalando su ejecución.

D

Deadlock (Interbloqueo): Una situación en la que dos o más procesos están bloqueados indefinidamente, esperando cada uno que el otro libere un recurso que necesita. Ninguno de los procesos puede avanzar.

G

Gestión de Archivos: La parte del sistema operativo encargada de organizar, almacenar, recuperar y manipular archivos y directorios (carpetas) en los dispositivos de almacenamiento. Esto incluye la creación, eliminación, copia, movimiento y control de acceso a los archivos.

Gestión de Memoria: La función del sistema operativo que asigna y desasigna espacio en la memoria RAM a los diferentes procesos y aplicaciones en ejecución. Su objetivo es optimizar el uso de la memoria, proteger los datos de un proceso de ser accedidos por otro y permitir que se ejecuten programas más grandes que la memoria física disponible (a través de la memoria virtual).

Gestión de Dispositivos (I/O Management): La función del sistema operativo que administra la comunicación entre el CPU y los dispositivos de entrada/salida (impresoras, teclados, ratones, discos duros, etc.). Permite que las aplicaciones usen estos dispositivos sin necesidad de conocer sus detalles de hardware.

K

Kernel (Núcleo): El componente central de un sistema operativo. Es la parte que siempre reside en la memoria RAM y tiene el control total de todo el sistema. Se encarga de gestionar los recursos de hardware, ejecutar procesos y proporcionar servicios a las aplicaciones.

M

Memoria Cache: Una pequeña y muy rápida área de memoria que almacena copias de los datos más frecuentemente utilizados del almacenamiento principal o de la RAM. Su objetivo es reducir el tiempo de acceso a los datos para el CPU.

Memoria Principal (RAM): La memoria RAM de la que ya hablamos en Laboratorio, pero en el contexto del sistema operativo es donde se cargan los programas y datos que están en uso activo para que el CPU pueda acceder a ellos rápidamente.

Memoria Virtual: Una técnica de gestión de memoria que permite al sistema operativo utilizar espacio en el disco duro como si fuera RAM adicional. Esto permite ejecutar programas más grandes que la memoria física disponible, aunque con un rendimiento más lento.

Multiprocesamiento: La capacidad de un sistema operativo para ejecutar múltiples procesos o hilos de ejecución en paralelo, utilizando múltiples CPU (o núcleos de CPU).

Multitarea: La capacidad de un sistema operativo para ejecutar varias tareas o procesos aparentemente al mismo tiempo. El SO alterna rápidamente entre ellos, dando la ilusión de concurrencia.

P

Proceso: Una instancia de un programa en ejecución. Cada proceso tiene su propio espacio de memoria, recursos y un estado de ejecución (listo, en ejecución, bloqueado).

S

Scheduler (Planificador): La parte del kernel del sistema operativo encargada de decidir qué proceso se ejecutará a continuación en el CPU y por cuánto tiempo. Su objetivo es optimizar el rendimiento y la equidad entre procesos.

Sistema de Archivos: La estructura lógica que el sistema operativo utiliza para organizar y gestionar los archivos en un dispositivo de almacenamiento (como un disco duro). Define cómo se almacenan, nombran y recuperan los archivos.

Sistema Operativo (SO/OS): El software más importante que se ejecuta en una computadora. Gestiona los recursos de hardware y software del equipo, proporciona servicios comunes para programas de aplicación y actúa como intermediario entre el usuario y el hardware. Ejemplos: Windows, macOS, Linux, Android.

T

Thread (Hilo de Ejecución): Una unidad de ejecución más pequeña dentro de un proceso. Un proceso puede tener varios hilos ejecutándose simultáneamente, compartiendo el mismo espacio de memoria y recursos del proceso.

V

Virtualización: La creación de una versión virtual de un recurso, como un sistema operativo, un servidor, un dispositivo de almacenamiento o un recurso de red. Permite ejecutar múltiples sistemas operativos o instancias de software en un solo hardware físico.
